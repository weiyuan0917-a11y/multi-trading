# 更新日志

本文件用于记录每次版本更新的时间戳和主要内容。后续每次优化、修复或发布客户版时，都需要追加一条记录。

## V1.0.24 - 2026-06-01 11:59:07 +08:00

- 优化客户端整体 UI 视觉层：统一深色交易终端背景、面板、按钮、输入框、表格和顶部栏的设计 token。
- 主操作按钮从蓝紫渐变调整为更克制的青绿色交易主色，降低页面色彩噪声，提升专业感和可读性。
- 导航栏选中态改为轻量边框和左侧强调线，卡片/面板圆角统一收敛到 8px，减少不同页面之间的视觉割裂。
- 增加全局焦点态、滚动条、表格悬停态和小屏间距优化，减少交互控件的突兀感。
- 同步客户版构建默认版本号为 `1.0.24`。

## V1.0.23 - 2026-06-01 10:52:01 +08:00

- 修复股票自动交易确认下单时 L3 `confirmation_token` 只校验进程环境变量，可能与设置页保存的当前用户 token 不一致而报“无效或缺失”的问题。
- L3 权限校验改为优先读取当前 owner 的 `data/user_env/<用户>.env`，并兼容无 owner 的 worker 历史路径，覆盖股票自动交易和三个期权自动交易下单入口。
- 自动交易页面的 L3 token 在当前浏览器会话内临时保留，确认下单时若为空会先提示输入，避免误发空 token。
- 统一自动交易模块的 L3 就绪状态改为按当前用户设置判断，减少期权自动交易页面状态误报。
- 同步客户版构建默认版本号为 `1.0.23`。

## V1.0.22 - 2026-06-01 10:14:18 +08:00

- 期权交易收益日历底部汇总口径调整：原“月累计”改为“本年累计”，按当前自然年从 1 月 1 日统计到今天。
- 保留“本月累计”独立展示，方便同时查看自然年收益和当前月份收益。
- 客户版前端构建改用独立 `.next-customer` 输出目录，并在客户构建时降低静态生成并发，避免开发端 `.next\dev` 缓存或 Windows build worker 崩溃阻塞客户版目录同步。
- 同步客户版构建默认版本号为 `1.0.22`。

## V1.0.21 - 2026-06-01 10:03:56 +08:00

- 期权交易收益日历纳入股票期权中长线 `stock_options_swing` 实盘 ledger 的真实提交成交，避免只靠 0DTE / 日内目录兜底。
- `order_detail` 成交详情扫描改为增量补充，即使已有 worker 日志或 history executions，也会继续补全缺失的期权成交。
- 增加收益日历回归测试，覆盖中长线 ledger 解析和 order_detail 增量补充。
- 同步客户版构建默认版本号为 `1.0.21`。

## V1.0.20 - 2026-06-01 09:19:22 +08:00

- 修复客户版飞书定时市场报告未把本地 owner 透传给后端 API，导致定时线程可能走系统默认账户/空凭证并降级为兜底行情的问题。
- 定时市场报告的宏观环境和美股板块轮动优先复用后端 `/market/analysis`、`/market/sectors` 链路，与 Dashboard 使用同一套 owner 账户行情上下文。
- 当后端 API 临时不可用时，仍保留飞书 Bot 进程内本地分析回退，避免报告完全发送失败。
- 增加客户版飞书运行时回归测试，覆盖 owner 透传和定时报告 API 优先链路。
- 同步客户版构建默认版本号为 `1.0.20`。

## V1.0.19 - 2026-05-31 18:18:36 +08:00

- 客户版启动器优先使用 Edge/Chrome App 模式打开 MultiTrading，窗口不显示浏览器地址栏和标签页，更接近桌面软件体验。
- 若本机未找到 Edge/Chrome 或 App 模式启动失败，会自动回退到系统默认浏览器打开。
- 支持通过 `MULTITRADING_BROWSER_APP_MODE=0` 关闭 App 模式，通过 `MULTITRADING_BROWSER_EXE` 指定浏览器路径。
- 同步开发启动器和备用客户启动器的窗口打开逻辑，保持入口行为一致。
- 同步客户版构建默认版本号为 `1.0.19`。

## V1.0.18 - 2026-05-31 15:24:06 +08:00

- 修复股票期权中长线交易 Worker 状态误判：PID 文件必须匹配真实 worker 命令行，避免陈旧 PID 被 Windows 复用后显示为“运行中”。
- 停止股票期权中长线 Worker 时会清理陈旧 PID 和 runtime 文件，不会终止无关进程。
- 启动/停止自动交易模块后立即清理状态缓存，减少页面刷新后继续显示旧状态的问题。
- 股票期权中长线交易页面新增“刷新持仓”按钮，可手动读取券商期权持仓快照并展示账户原有未托管持仓。
- 手动刷新持仓快照不会启动 Worker，也不会提交订单；仅更新页面展示所需的 runtime 快照。
- 同步客户版构建默认版本号为 `1.0.18`。

- 客户版运行环境显式关闭 API 启动时自动恢复股票自动交易 Worker。
