# MultiTrading 系统使用手册

本文档基于当前项目的前端页面、后端接口和导航结构编写，面向日常使用者说明系统功能、操作流程和界面入口。全文按“系统功能简介、系统使用流程、界面展示”三个模块组织。

## 一、系统功能简介

### 1.1 系统定位

MultiTrading 是一套面向多账户、多策略场景的智能交易工作台。系统以本地 Agent 为核心，集成了行情查看、研究分析、策略回测、手动交易、自动交易、通知推送和账户配置等能力，适合用于交易研究、参数验证、执行监控和日常运维。

### 1.2 核心功能模块

#### 1.2.1 首次配置与系统接入

系统提供 `本地配置向导`、`首次配置 Setup` 和 `配置中心` 页面，用于完成以下初始化工作：

- 绑定本地 Agent 与当前登录用户
- 配置券商账户及 API 凭证
- 配置 LLM、TradingAgents、OpenBB、A 股研究数据源
- 配置飞书通知、MCP 授权与个人 API Key
- 配置交易费用模型、风控参数和后台服务启停

该模块是整个系统的入口，决定后续研究、回测、交易和自动化功能能否正常使用。

#### 1.2.2 总览与市场观察

系统提供 `总览 Dashboard`、`市场分析`、`新闻信息流` 和 `信号中心` 等页面，用于快速掌握市场状态：

- `总览 Dashboard`：查看关键指标、跨市场行情、板块轮动和整体市场环境
- `市场分析`：从区域市场、宽度变化、数据源状态等角度进行综合观察
- `新闻信息流`：聚合国际、国内及持仓相关新闻，并附带情绪标签
- `信号中心`：对多个标的进行技术指标和触发状态监控，如 RSI、均线、底部反转提示等

该模块主要服务于盘前观察、盘中监控和策略机会筛选。

#### 1.2.3 研究与智能分析

系统提供 `研究中心`、`TradingAgents 智能体` 和 `Agent Strategy Lab` 三类研究入口：

- `研究中心`：运行 Research、策略矩阵、ML 矩阵和 A/B 报告，研究参数可与 Auto Trader 当前配置对齐
- `TradingAgents 智能体`：围绕单一标的发起多智能体分析，输出研究报告、结论、置信度和风险提示
- `Agent Strategy Lab`：生成候选参数、执行验证回测、查看审批状态，仅产出配置草稿，不直接触发实盘下单

该模块主要用于形成研究结论、寻找策略参数、比较候选方案，并为后续回测和自动交易做准备。

#### 1.2.4 回测与策略验证

系统提供 `回测中心` 和策略专项验证页面，用于验证交易逻辑和参数效果：

- `回测中心`：支持多策略对比、收益回撤分析、交易明细查看和服务器 K 线缓存回测
- `QQQ 0DTE 自动期权交易` 页面中的 `策略验证工作台`：支持参数矩阵、OPRA 解析、快照写入、策略回测和高级 JSON 配置
- `股票期权日内交易`、`股票期权中长线交易` 等专项页：承接对应自动交易模块的验证和配置

回测模块的主要目标是验证策略在不同周期、不同参数和不同数据范围下的表现，为实盘启用提供依据。

#### 1.2.5 手动交易与期权交易

系统提供 `交易面板` 和 `期权交易` 页面，支持人工执行交易操作：

- `交易面板`：下单、查看持仓、查看订单、撤单、查看账户和风控状态
- `期权交易`：查询到期日、期权链、组合腿、费用试算、期权订单、期权持仓和组合回测

该模块适合用于人工验证交易链路、进行单笔操作或在自动交易前做执行确认。

#### 1.2.6 自动交易

系统提供统一的 `自动交易` 总入口，并拆分为四个二级模块：

- `股票自动交易`
- `期权 0DTE`
- `股票期权日内交易`
- `股票期权中长线交易`

自动交易模块支持：

- 查看各模块运行状态、更新时间、错误信息
- 启动或停止单个模块，也可统一启停
- 管理研究参数、风控参数、信号执行方式和 Worker 运行状态
- 区分半自动与全自动两种执行模式

其中，股票自动交易更偏向强势股扫描、策略评分和自动执行；期权自动交易更偏向专项策略、快照同步和专用 Worker 管理。

#### 1.2.7 通知与运维

系统提供 `通知中心`、`个人中心`、`订购/升级` 等页面，用于保障系统连续运行：

- `通知中心`：管理飞书机器人、自动调度、通知偏好和监控条件
- `个人中心`：查看本地绑定、权限等级、本地授权 License 和安全设置
- `订购/升级`：查看套餐、付款信息与功能权限范围

该模块主要服务于系统运行维护、权限管理和消息触达。

### 1.3 系统特点

与传统单一交易页面相比，本系统具备以下特点：

- 以“配置 -> 研究 -> 回测 -> 执行 -> 通知”作为完整闭环
- 同时覆盖股票、期权、短线和中长线场景
- 支持多智能体研究、参数矩阵和配置草稿审批
- 支持本地 Agent 架构，便于控制凭证、账户和实盘执行边界
- 自动交易与手动交易并存，便于渐进式上线

## 二、系统使用流程

### 2.1 首次使用流程

首次进入系统时，建议按照以下顺序完成配置：

1. 进入 `账户登录 / 注册` 页面，完成系统账号登录。
2. 进入 `本地配置向导` 或 `首次配置向导`，确认本地 Agent 在线并完成本地身份绑定。
3. 进入 `配置中心`，优先完成 `账户与券商` 配置，填写券商凭证并注册默认账户。
4. 在 `密钥配置` 中补充 Broker、Feishu、LLM 和市场数据相关 Key。
5. 在 `Research 数据源` 和 `TradingAgents` 区域配置研究能力，如 OpenBB、A 股数据源、LLM Provider 与模型。
6. 在 `费用模型` 中确认当前默认账户对应的费率模板。
7. 在 `风控与服务` 中设置最大订单金额、单日损失阈值等参数，并启动相关服务。
8. 完成后进入 `总览 Dashboard` 检查系统是否能够正常返回行情和摘要数据。

### 2.2 日常研究流程

在进行策略研究和交易准备时，建议采用以下流程：

1. 打开 `总览 Dashboard`、`市场分析` 和 `新闻信息流`，先判断市场环境、热点方向和风险偏好。
2. 在 `信号中心` 维护重点观察标的，查看技术指标与触发状态。
3. 如需深度研究，进入 `TradingAgents 智能体`，围绕目标标的发起问题并获取研究报告。
4. 如需参数筛选，进入 `研究中心` 运行 Research、策略矩阵或 ML 矩阵。
5. 如需形成候选配置，进入 `Agent Strategy Lab` 生成候选参数并查看验证结果。

这一流程适合盘前准备、周度策略迭代和特定标的研究。

### 2.3 回测验证流程

当研究结论形成后，应先进行回测验证，再进入实盘执行：

1. 进入 `回测中心`，选择标的、回测周期、策略范围和 K 线参数。
2. 如需提高稳定性，可先下载 K 线到服务器，再勾选“回测使用服务器已下载的 K 线”。
3. 点击“运行回测”，查看收益对比、回撤、胜率、交易数和交易明细。
4. 如是 QQQ 0DTE 或专项期权策略，进入对应策略页的 `策略验证工作台`，运行参数矩阵、快照保存和专项回测。
5. 对比不同参数组合后，选定更优方案，为手动交易或自动交易提供依据。

### 2.4 手动交易流程

当需要人工执行股票或期权交易时，可按以下步骤操作：

1. 进入 `交易面板` 或 `期权交易` 页面。
2. 选择默认账户或目标账户，确认账户状态和持仓信息正常。
3. 填写交易标的、方向、数量和价格等参数。
4. 提交订单前，核对风险参数、账户可用状态和价格信息。
5. 提交后，在订单列表中查看状态；如有需要，可在列表中执行撤单。

该流程适用于人工验证链路、半自动确认或独立手工交易场景。

### 2.5 自动交易上线流程

自动交易建议采用“先研究、再回测、后上线、持续监控”的方式：

1. 在 `研究中心`、`TradingAgents` 或 `Agent Strategy Lab` 中完成参数准备。
2. 在 `回测中心` 或专项策略验证页中完成效果验证。
3. 进入 `自动交易` 总入口，选择目标模块，如 `股票自动交易`、`期权 0DTE`、`股票期权日内交易` 或 `股票期权中长线交易`。
4. 检查模块的运行状态、风控模式、参数配置和 Worker 状态。
5. 根据需要选择半自动或全自动模式。
6. 启动对应模块，观察是否正常产生日志、信号和执行反馈。
7. 在运行期间持续查看 `通知中心`、`自动交易` 模块页和相关策略页，确认无异常报错。

### 2.6 通知与复盘流程

系统运行后，建议使用以下方式做持续跟踪：

1. 在 `通知中心` 启动飞书机器人和自动调度。
2. 根据需要配置底部反转监控、自定义标的列表和通知偏好。
3. 通过 `总览 Dashboard`、`自动交易` 和 `通知中心` 定期检查运行状态。
4. 在回测中心、研究中心和自动交易模块中复盘信号来源、参数变化和执行结果。
5. 在 `个人中心` 中定期检查授权状态与套餐权限，避免因权限变化影响功能使用。

## 三、界面展示

### 3.1 整体导航结构

系统采用左侧导航栏作为主入口，主要菜单包括：

- 本地 Agent 向导
- 首次配置 Setup
- 总览 Dashboard
- 市场分析
- 新闻信息流
- 信号中心
- 回测中心
- 自动交易
- 研究中心
- TradingAgents 智能体
- Agent Strategy Lab
- 交易面板
- 期权交易
- 通知中心
- 订购与升级
- 个人中心及管理页

用户可以按照“先配置、后研究、再执行”的思路，在不同页面间切换。

### 3.2 配置类界面

#### 3.2.1 本地配置向导 / 首次配置向导

该页面用于首次接入，界面通常包含：

- 当前步骤导航
- 本地 Agent 检查状态
- 用户与 owner 绑定信息
- 券商配置、LLM 配置、通知配置和研究数据源配置项

适合第一次部署系统或为新账号重新初始化环境时使用。

#### 3.2.2 配置中心

`配置中心` 是日常维护的核心页面，主要分区包括：

- 账户与券商
- 密钥配置
- Research 数据源
- TradingAgents
- 费用模型
- 风控与服务
- 高级设置

页面以卡片或分区方式展示当前状态、下一步建议和保存按钮，适合做集中配置管理。

### 3.3 市场观察类界面

#### 3.3.1 总览 Dashboard

`总览 Dashboard` 页面主要展示：

- 关键指标
- 跨市场行情
- 板块轮动
- 数据状态提示

适合快速了解系统当前抓取到的市场摘要，是登录后的高频入口。

#### 3.3.2 市场分析

`市场分析` 页面进一步展开市场状态，通常包括：

- 不同市场分组数据
- 市场宽度和涨跌结构
- 数据源说明
- 区域市场观察建议

适合盘前、盘中判断整体环境和节奏。

#### 3.3.3 新闻信息流

`新闻信息流` 页面主要展示：

- 国际、国内和持仓相关新闻
- 新闻来源、发布时间和情绪标签
- 利多、利空、中性分类筛选

适合用于事件驱动观察和持仓新闻跟踪。

#### 3.3.4 信号中心

`信号中心` 页面主要包含：

- 标的列表维护
- 最新价格与技术指标快照
- RSI、均线、底部反转等信号状态
- 手动刷新能力

适合作为盘中监控面板使用。

### 3.4 研究与回测类界面

#### 3.4.1 研究中心

`研究中心` 页面主要展示：

- Research 面板
- 策略矩阵入口
- ML 矩阵结果
- A/B 报告结果

适合做股票池研究、参数比较和结果汇总。

#### 3.4.2 TradingAgents 智能体

`TradingAgents 智能体` 页面主要展示：

- 会话区
- 提问模板
- 任务进度
- 多智能体事件流
- 最终研究报告

适合围绕单一标的做深度问答式研究。

#### 3.4.3 Agent Strategy Lab

`Agent Strategy Lab` 页面主要展示：

- 研究配置
- 运行状态
- 数据质量
- 审批状态
- 候选参数与验证结果

该页面定位为“候选生成与审批工作台”，不会直接触发下单。

#### 3.4.4 回测中心

`回测中心` 页面主要展示：

- 回测参数输入区
- K 线缓存相关选项
- 运行按钮和进度状态
- 策略收益对比图
- 策略指标明细
- 交易明细查看和导出

适合进行多策略、多周期的统一回测比较。

### 3.5 交易执行类界面

#### 3.5.1 交易面板

`交易面板` 页面主要包含：

- 下单表单
- 账户概览
- 持仓列表
- 订单列表
- 风控状态

适合股票类手动下单和撤单。

#### 3.5.2 期权交易

`期权交易` 页面主要包含：

- 到期日和期权链查询
- 组合腿配置
- 费用试算
- 期权订单与持仓
- 组合回测和盈亏日历

适合期权结构研究和期权人工交易。

### 3.6 自动交易类界面

#### 3.6.1 自动交易总入口

`自动交易` 首页是统一运行总览，主要展示：

- 各模块运行状态
- 是否有模块正在运行
- 全部启动 / 全部停止按钮
- 统一风控提醒
- 各子模块的进入入口

适合作为自动交易运维总控台使用。

#### 3.6.2 股票自动交易

`股票自动交易` 页面通常包含：

- 今日交易摘要
- 交易模式说明
- 策略参与评分配置
- 研究中心联动入口
- 启动 Worker、保存配置和模板应用等操作

适合日常股票自动交易管理。

#### 3.6.3 期权专项自动交易页面

期权专项页面包括 `期权 0DTE`、`股票期权日内交易`、`股票期权中长线交易` 等入口，主要展示：

- 实盘自动交易面板
- Worker 状态与启停按钮
- 快照同步
- 参数表单
- 高级 JSON 配置
- 策略验证工作台

其中 `QQQ 0DTE 自动期权交易` 页面还将 `实盘自动交易`、`Agent Strategy Lab` 和 `高级调试/手动回测` 集中在一个页面内，便于专项策略从研究到上线的闭环管理。

### 3.7 通知与个人管理界面

#### 3.7.1 通知中心

`通知中心` 页面主要展示：

- 飞书配置与机器人状态
- 自动调度状态
- 通知偏好
- 底部反转监控条件
- 标的导入与保存操作

适合做通知链路维护和监控项管理。

#### 3.7.2 个人中心

`个人中心` 页面主要展示：

- 登录身份
- 本地绑定
- 权限等级
- 本地授权 License
- 安全设置

适合查看账号状态与功能权限。

#### 3.7.3 订购 / 升级

`订购 / 升级` 页面主要展示：

- 套餐方案
- 功能差异
- 付款信息

适合查看当前套餐和后续升级需求。

## 结语

本系统不是单一交易页，而是一套覆盖“配置、观察、研究、回测、执行、通知”的完整交易工作台。实际使用中，建议始终按照“先完成配置、再做研究验证、最后启用自动执行”的顺序操作，以降低实盘风险并提高系统稳定性。
